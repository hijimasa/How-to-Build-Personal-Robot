# How to Build Personal Robot — コンセプト方針書

本書はチュートリアルサイト「How to Build Personal Robot」の編集方針を定める 1 枚もののコンセプト資料である。各章の執筆・分業に入る前に、本書で「対象読者」「スコープ」「トーン」「安全啓発の扱い」を揃えることを目的とする。

章ごとに前提知識・語り口・安全喚起の濃さがズレると、初心者読者は一気に迷子になる。本書は **「各章執筆者（人間・エージェント問わず）が最初に必ず読む共通ルールブック」** として運用する。

---

## 1. プロジェクトの目的

「小中学校の理科の授業しか覚えていないが、自分でロボットを作ってみたい」個人開発者が、**部品を焼損させずに・自分でデータシートを読みながら**、最終的に「自作の動くロボット」を一台完成させられるようになることをゴールとする。

既存のロボット入門書・ブログ記事は、以下のいずれかに偏りがちである。

- 「コードを写経すれば動く」式で、電気の基礎・なぜその抵抗値なのか・なぜそのトランジスタなのかを説明しない
- 逆に電子工学の教科書的で、読者が挫折するまで「動くもの」が出てこない

本チュートリアルは、**「動くものを早く出す」 × 「データシートの読み方を最初から教える」** の両立を目指す。

---

## 2. ターゲット読者像（ペルソナ）

### 想定する「できること」
- パソコンの基本操作（OS インストール、エディタでの文字入力、ターミナルでコマンド実行）
- 中学理科レベルの知識：オームの法則（V=IR）は見れば思い出せる、直列・並列の違いはなんとなくわかる
- Python または C のサンプルコードを **写経して動かす** ことはできる
- はんだごての使用経験は問わない（章の中で教える）

### 想定する「できないこと／知らないこと」
- オシロスコープ・ロジックアナライザは持っていない（テスタは持っている or 買ってもらう前提）
- データシートを見たことがない、あるいは見ても「どこを見ればいいか」がわからない
- 「プルアップ／プルダウン」「オープンドレイン」「ソース／シンク電流」という用語を知らない
- 3.3V と 5V の混在で何が起きるか、直感的にはわからない
- トランジスタ／MOSFET を「スイッチとして使う」という発想がない

### 想定しない読者
- 大学で電子工学を専攻した人（本書は物足りないはず）
- すでに Arduino で LED チカチカができて、次に何を学ぶか迷っている中級者（対象にしてもよいが、最優先ではない）

> **ルール**：各章は「上記ペルソナが読んで理解できるか」を唯一の合格基準とする。章の途中で未定義の用語が出たら、その章内で定義するか、既出章へのリンクで解決する。

## 3. スコープ（扱う / 扱わない）

### 扱う
- 直流モータ（ブラシ付き）、サーボモータ、ステッピングモータの駆動
- LED（高輝度含む）の点灯と、電流制限の考え方
- 基本センサ：スイッチ、ポテンショメータ、測距センサ、IMU、エンコーダ
- マイコン：Arduino 系（Uno / Nano / R4 など）と Raspberry Pi 系の両方を扱う（§6 参照）
- 電源設計の初歩：電池・AC アダプタ・DCDC、ロジック電源とモータ電源の分離
- データシート読解：絶対最大定格、推奨動作条件、電気的特性、ピン配置、タイミングチャート
- ごく簡単な制御：On/Off、PWM、ブロッキング回避、割り込み、最低限の P 制御

### 扱わない（少なくとも v1 では）
- ROS/ROS 2 を使った本格的な自律移動（別サイト／別シリーズに委ねる）
- 基板設計（KiCad など）：ブレッドボード＋ユニバーサル基板までを範囲とする
- 機械設計（CAD／3D プリンタ）：本題から外れるため、紹介にとどめる
- 高電圧・AC 系（感電・発火リスクが高く、初心者向けで扱うのは不適切）

## 4. 安全ファースト方針

本書の最大の差別化要素。「焼損事故で心が折れる前に、焼損事故を未然に防ぐ知識を与える」ことを核にする。

### 方針
- **第 1 章を安全・故障・計測に充てる**。LED チカチカより先に「電流制限抵抗がないと何が起きるか」「GPIO に直接モータをつなぐと何が起きるか」を **NG 事例＋データシート根拠** で見せる。
- 各章の冒頭に **「この章で壊しやすいもの」欄** を置く。具体的な型番と失敗パターンを明示。
- 各章末に **「動作確認チェックリスト」** を置く（例：電源投入前にテスタで短絡確認したか、論理電圧を間違えていないか、など）。
- 危険な作業（リポバッテリ充電、はんだごて、高輝度 LED の直視）は Material for MkDocs の `!!! danger` admonition で強調する。注意喚起の重複を恐れない。

### トーン
- 「脅す」ではなく「根拠を示す」。「危ないからやめろ」ではなく「データシートの 5 ページに絶対最大定格 40mA と書いてある、君は今これを超えようとしている」と書く。
- 読者を子ども扱いしない。大人の個人開発者が相手なので、淡々と事実とリスクを伝える。

## 5. データシート読解の扱い

**独立章で基礎を教える × 各部品登場章で実例として再掲する**、の二段構え。

### 独立章（例：第 2 章「データシートの読み方」）
- データシートの典型的な章構成（Features / Absolute Maximum Ratings / Recommended Operating Conditions / Electrical Characteristics / Pin Configuration / Timing Diagrams / Package）
- 必ず最初に見るべき項目：**絶対最大定格、推奨動作条件、I/O 電流の吸込み／吐出し能力**
- 記号の読み方：V_IH / V_IL、I_OH / I_OL、V_CC、V_DD、T_A、T_J
- 「typ. / min. / max.」の意味と、min/max を採用すべき場面
- データシート例：抵抗・LED・Arduino Uno の ATmega328P・汎用 NPN トランジスタ・L298N のような定番モータドライバ

### 各部品登場章での再掲
- 新しい部品を使うときは必ずデータシート該当ページのスクリーンショット or 引用を載せ、**「この値をなぜ採用したか」を読者と一緒に読み解く** 形式にする。
- データシート URL は必ず本文中にリンクで示す（リンク切れ対策として、アーカイブ URL も可能な限り併記する — 要検討）。

## 6. 教材ボード選定

### 基本思想：「やりたいことファースト」で選ぶ

個人開発者は「Arduino で基礎を学んでから RasPi に卒業する」という段階を踏まない。最初から自分の**作りたいもの**を起点にボードを選ぶ。本書はこの前提を受け入れ、「まず選定ガイドで適切なボードに誘導する → それぞれに対して安全に動かせる基礎を授ける」という構成を取る。

### v1 対象ボード（2 トラック制）

本書のコアカリキュラム（電気基礎・データシート読解・焼損回避）が**最大限活きるメイントラック**と、目的が明確に異なる**別トラック**に分ける。

#### メイントラック：裸のマイコン（ブレッドボード前提）

本書は **動作保証のために代表機種を特定型番で指定する**（§11 の編集方針参照）。メイン代表と補欠を明示する。

| ボード | 代表機種（メイン） | 補欠（章末コラム扱い） | ロジック電圧 | 書き込み方式 | 無線 |
|---|---|---|---|---|---|
| **Arduino Uno** | **Uno R3**（公式品、秋月・Switch Science で通年在庫） | Uno R4 Minima（USB-C・モダン化したくなった時） | 5V | Arduino IDE + シリアル（CH340/FT232RL 内蔵） | なし |
| **Raspberry Pi Pico** | **Raspberry Pi Pico（無印）** | Pico 2（RP2350）、Pico W（Wi-Fi） | 3.3V | UF2 ドラッグ&ドロップ、Arduino IDE、MicroPython | Pico W のみ Wi-Fi |
| **ESP32 DevKit** | **ESP32-S3-DevKitC-1**（Espressif 公式、ネイティブ USB JTAG/CDC でドライバ不要） | ESP32-WROOM-32 ベース DevKitC-V4（既存チュートリアル豊富だが CH340/CP2102 ドライバ必要） | 3.3V | Arduino IDE、ESP-IDF、MicroPython | Wi-Fi / BT 標準 |

**Arduino Uno が R3 primary の理由**：公式品・クローンとも流通量が圧倒的で、既存チュートリアル・書籍・ライブラリ事例の大半が R3 前提。読者がエラーで詰まった時の検索ヒット率が段違いに高い。また 5V 純粋な ATmega328P は **5V / 3.3V 混在事故を教える教材として切れ味が良い**（R4 は内部 3.3V + 5V トレラントで挙動が複雑）。

**ESP32 が S3-DevKitC-1 primary の理由**：初心者の最大の躓きポイントである USB シリアルドライバ（CH340 / CP2102）の導入が不要（ネイティブ USB）。Espressif 公式現行品で Switch Science・秋月に通年在庫。

派生として **Seeed XIAO RP2040 / XIAO ESP32-S3** を「同じ MCU の超小型版」として随所で紹介（独立トラックにはしない）。

#### 別トラック：プラットフォーム系（ハード設計を最小化）

| ボード | 代表機種（メイン） | 補欠 | 位置づけ | 主な用途 |
|---|---|---|---|---|
| **Raspberry Pi** | **Raspberry Pi 4 Model B（4GB or 8GB）** | Pi 5（新しく性能が必要になった時）、Zero 2 W（小型化） | Linux SBC。GPIO は 3.3V、電流駆動能力は小さい | カメラ、画像処理、ROS 2、Python リッチなアプリ |
| **M5Stack** | **M5Stack Core2** | StickC Plus2（小型・ウェアラブル用途）、AtomS3（超小型） | 画面・電池・ボタン・スピーカ・IMU 込みの完成品ガジェット | ハード設計を回避してソフトで遊びたい、表示ありのガジェット |

**RasPi が Pi 4 primary の理由**：Pi 5 は性能は高いが発熱・消費電力・アクティブクーラ必須など初心者のハードルが上がる。Pi 4 は依然として入手性が高く、HAT ・ケース・書籍などエコシステムの厚みも段違い。ROS 2 Humble/Jazzy の動作実績も豊富。

**M5Stack が Core2 primary の理由**：画面サイズ・バッテリ容量・Grove コネクタ数・作例の豊富さで入門向けとしてバランスが良い。StickC Plus2 は小さすぎて初学者がハンダ付け・配線しづらい場面が出る。

### なぜこの 2 トラック分けか

- **裸のマイコン**は、読者が**自分で電子回路を組む**ことを前提にするため、本書の「焼損回避」「データシート読解」の価値が最も高い。
- **RasPi (SBC)** は Linux・カメラ・ネット連携という異質な動機で選ばれる。GPIO は 3.3V・駆動能力が小さく、直結事故の典型例が違う（USB 電源容量、SD カード破損など独自トピックあり）。
- **M5Stack** は「既に完成したハードの上でソフトを書く」プラットフォーム。Grove モジュールで部品を買えば焼損リスクが大きく下がるため、電子工作章の大半をスキップできる。ただし**それでも**データシート読解・モータ駆動原理の理解は価値があるので、橋渡し節で接続する。

### 選定ガイド章（独立章として配置）

読者が最初に通る章。**3 段階のフローチャート**で誘導する。

1. **最上位分岐**：あなたは何をしたい人か？
   - ハードから組み立てたい／電子工作も学びたい → メイントラック
   - カメラ・画像処理・ネット連携・Linux で動くアプリを作りたい → RasPi (SBC)
   - ハード設計は最小限、ソフトで遊びたい／画面ある方がやる気出る → M5Stack
2. **メイントラック内の分岐**：
   - 無線（Wi-Fi / BT）が要るか → Yes → ESP32 DevKit / No → 下へ
   - 5V の古典 IC（74HC シリーズ等）を使いたい、壊れにくさ重視 → Arduino Uno
   - とにかく安く・モダン・USB-C・小型 → Raspberry Pi Pico
3. **サイズ軸**：完成させたものを小型化したい → XIAO 系へ派生

### トラック横断で最初に教えること（§4 の安全啓発と結合）

どのトラックを選んでも、次の 4 項目は**最初に**読ませる。

- ロジック電圧（5V / 3.3V）の違いと混在時の事故
- GPIO の吸込み／吐出し電流の上限とその根拠（データシート該当ページ）
- PWM のハード実装／ソフト実装の違いとジッタ
- リアルタイム性（ベアメタル MCU ⇄ Linux SBC）

### 要検討
- 各ボードの「Hello 章」の分量上限（各 4〜6 ページを目安と仮置き）

## 7. 「壊れる実例」の見せ方（章テンプレ）

各ハンズオン章は以下のテンプレートに従う。

1. **動機**：この章で作るもの（1〜2 行）
2. **この章で壊しやすいもの**：具体的な型番と失敗パターン
3. **素朴な（NG）回路／コード** を先に提示
4. **なぜダメか** をデータシートの該当箇所を引用して解説
5. **正しい回路／コード** の提示と、変更点の根拠（抵抗値・トランジスタ選定理由など）
6. **動作確認チェックリスト**（テスタで何を測るか、電源投入順序）
7. **よくあるトラブル FAQ**（煙が出た／動かない／熱い／暴走した）
8. **次章への橋渡し**

このテンプレは「章テンプレ」として別ファイル（`docs/_templates/chapter_template.md` 等）に切り出し、分業時は必ずこれを先に埋める運用にする。

## 8. 最終成果物（卒業課題）

読者が本チュートリアルを完走したときに作れるようになっているもの。

### 候補（要検討 — 最終的には 1〜2 個に絞る）

A が最も敷居が低く、電源・モータドライバ・センサ・制御を一通りカバーできるため第一候補。

## 10. 章構成ドラフト（v1）

§6 の 2 トラック制を受けて、構成を **「共通基礎 → 選定ガイド → 各トラックの Hello → 共通トピック章（差分表付き） → トラック別プロジェクト」** の層構造にする。

### Part I : 共通基礎（全トラック必読）

1. **はじめに** — 本書の読み方／対象読者／必要な工具・テスタ・基本部品
2. **壊さないための基礎** — オームの法則の復習、絶対最大定格、電圧と電流の直感、GPIO 直結の危険性、テスタの使い方
3. **データシートの読み方** — データシート独立章（§5）
4. **電源の基礎** — 電池・AC アダプタ・DCDC、ロジック電源とモータ電源の分離、USB 給電の落とし穴

### Part II : マイコン選定ガイド

5. **自分に合ったマイコンを選ぶ** — §6 の選定フローチャート。ここで読者は自分のトラックを決める

### Part III : 各ボードの Hello 章（該当トラックを読む）

ここは読者が**全部読む必要はない**。選んだボードの章だけを読む構成にする。

6. **Arduino Uno を使い始める** — IDE、シリアル、L チカ、5V の扱い
7. **Raspberry Pi Pico を使い始める** — UF2 書き込み、Arduino IDE / MicroPython、3.3V の扱い
8. **ESP32 DevKit を使い始める** — Arduino core on ESP32、電源容量、技適
9. **Raspberry Pi を使い始める** — Linux セットアップ、SSH、`gpiozero`、3.3V GPIO の駆動能力
10. **M5Stack を使い始める** — UIFlow / Arduino、画面描画、Grove、内蔵バッテリの扱い

### Part IV : 共通トピック章（代表ボードで作例 + 移植差分表）

**代表ボードのデフォルトは Arduino Uno R3**。以下の章はすべて Uno R3 で動作検証した作例を本文に載せる。各章末に移植ポイント差分表（§11）を付けて他ボードへの橋渡しを担保する。

**例外ルール**：章の題材が ESP32 でないと実装が成立しない（Wi-Fi / BLE / 高速 ADC / 大量 PWM チャネル等）場合のみ ESP32-S3-DevKitC-1 を代表に変更する。現時点の章立て（下記 11〜18）はすべて Uno R3 で成立するため、**v1 開始時は全章 Uno R3 デフォルト**で走る。

11. **LED を正しく光らせる** — 電流制限抵抗、VF、高輝度 LED の注意（代表：Uno R3）
12. **スイッチ入力** — プルアップ／プルダウン、オープン入力の危険性、チャタリング（代表：Uno R3）
13. **トランジスタ／MOSFET をスイッチとして使う** — GPIO で大電流を扱う、ゲート抵抗、ロジックレベル MOSFET の選び方（代表：Uno R3）
14. **DC モータを回す** — モータドライバ IC（DRV8835 / TB67H450 / L298N 比較）、逆起電力、電源分離（代表：Uno R3）
15. **PWM と速度制御** — ハードウェア PWM vs ソフトウェア PWM、周波数の選び方（代表：Uno R3）
16. **サーボモータ** — PWM 信号、電源容量、ブラウンアウト対策（代表：Uno R3）
17. **センサ入門** — アナログ入力、I2C、SPI の選び方、5V/3.3V 混在時のレベル変換（代表：Uno R3）
18. **（任意）ステッピングモータ** — A4988 / DRV8825、マイクロステップ、電流制限の設定（代表：Uno R3）

### Part V : トラック別プロジェクト章

19. **プロジェクト A（メイントラック向け）：ライントレースカーを作る** — Arduino Uno ベース、全基礎の統合
20. **プロジェクト B（RasPi トラック向け）：カメラつきローバーを作る** — Raspberry Pi + モータドライバ HAT、OpenCV の初歩
21. **プロジェクト C（M5Stack トラック向け）：Grove モジュールで作るセンサ工作** — 画面・無線・バッテリを活かした題材

## 11. 執筆ルール（分業時の共通仕様）

### 動作保証の編集方針（最重要）

本書はハード絡みの入門書であり、**「この通りに組めば動く」という再現性が信頼の生命線**である。したがって次の方針を採る。

- **作例は必ず公式品・正規流通品の特定型番で検証し、その型番を本文に明記する**（例：「Arduino Uno R3（公式品）」「ESP32-S3-DevKitC-1（Espressif 公式）」）。
- **互換品・クローン品についての立場**：「存在は認識しており、読者が選ぶことは妨げない。ただし品質にばらつきがあり、本書の動作は公式品でしか保証しない」という**中立＋非推奨スタンス**で統一する。
- クローン品固有のトラブル（USB シリアルドライバの差、電源レギュレータ品質、偽造チップによる書き込み不能など）は**付録**に集約し、本編では誘導リンクのみ置く。本編が「クローン対応の条件分岐だらけ」になるのを防ぐため。
- 「クローンで動かない」という読者報告は、本編ではなく付録 FAQ 側で受ける運用にする。

### 文体
- 「です・ます」調（敬体）。読者を見下さない。
- 専門用語は初出時に必ずカッコ書きで簡単な定義を添える。
- 英語表記は原則カタカナ併記（例：「ソース電流（source current）」）。

### 図版・回路図
- **回路図ツールは Fritzing を採用**。初心者が後で自分でも描ける敷居の低さ、日本語環境での安定動作、ブレッドボード図の視認性を評価。ただし次の**緩和策を必須**とする（Fritzing のブレッドボード図は物理配置のイメージに読者を引きずり、NG 配線を覚える副作用があるため）。
  1. **ブレッドボード図とシェマティック（回路図）図を必ず併記**する。Fritzing はビュー切替で両方出力できるので、両方を書き出して章内に並べる。
  2. 各章テンプレに **「この配線は一例です。同じ回路図が成立すれば物理配置は自由です」** の注記を定型で入れる。可能であれば 1 つの回路図に対して複数の物理配置例を見せる付録章を v1 に入れる。
  3. **本書で使う部品型番は独自ライブラリ化**する（Fritzing 既存ライブラリの部品は実物の型番・ピン配置と一致しないことがあるため）。独自ライブラリは `docs/_assets/fritzing/` に置き、版管理する。
- 写真は実機写真を推奨。完成形だけでなく「壊した／煙が出た」写真があれば貴重なので積極採用。

### コード
- **Arduino Uno / Pico / ESP32 DevKit**：Arduino 言語（C++ ベース）を第一選択にする（共通スケッチが流用できるため）。必要に応じて **MicroPython** / **ESP-IDF** / **C SDK** を併記。
- **Raspberry Pi**：Python 3（`gpiozero` → 必要に応じて `RPi.GPIO` や `lgpio`）。
- **M5Stack**：Arduino 言語をデフォルト。UIFlow（Blockly）は付録扱い。
- 写経可能な **完全動作コード** を載せる。部分抜粋にとどめない。コード末尾に GitHub 上のリンクを置く。

### 移植ポイント差分表（共通トピック章で必須）

Part IV の共通トピック章は、**代表 1〜2 ボードで作例を示した上で**、末尾に必ず「移植ポイント差分表」を置く。テンプレート：

| 項目 | Arduino Uno | Raspberry Pi Pico | ESP32 DevKit | Raspberry Pi | M5Stack |
|---|---|---|---|---|---|
| ロジック電圧 | 5V | 3.3V | 3.3V | 3.3V | 3.3V |
| 使用ピン例 | D9 | GP15 | GPIO18 | BCM18 | G26 |
| 電流駆動上限（データシート記載値） | — | — | — | — | — |
| 代表的な API 例 | `digitalWrite(9, HIGH)` | `gpio_put(15, 1)` | `digitalWrite(18, HIGH)` | `LED(18).on()` | `digitalWrite(26, HIGH)` |
| 章内で必要な追加部品 | — | — | — | — | Grove ブレークアウト等 |
| 注意点（あれば） | — | — | — | — | — |

- 全セルを埋める必要はない。**代表ボード以外は「該当ピン・API・注意点」の 3 項目を最低限埋める**。
- 差分表は読者の**パニック時のレスキュー表**として機能する。「作例と自分のボードが違う、どうすれば」という最頻出の詰まりをここで解消する。

### admonition 運用
- `!!! danger`：人体・機材に即座にダメージが出るもの（感電、発火、失明、リポバッテリ発火）
- `!!! warning`：部品焼損・誤動作の可能性
- `!!! note`：補足・豆知識
- `!!! tip`：作業を楽にする Tips
- `!!! info "ボード差分"`：差分表への導線、または短い移植注記（カスタム admonition として定義）
